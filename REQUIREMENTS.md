### 模块3: 冲击事件概率模型（希尔伯特相位分析）

#### 功能描述
基于希尔伯特变换的相位分析，预测冲突升级的时间窗口。

#### 核心功能
| 功能点 | 优先级 | 复杂度 |
|--------|--------|--------|
| 相位图可视化（复平面） | P0 | 高 |
| 时间序列相位图（30天） | P0 | 高 |
| 希尔伯特变换计算 | P0 | 高 |
| 峰值预测算法 | P0 | 高 |
| 振幅强度分析 | P1 | 中 |
| 相位速度计算 | P1 | 中 |
| 周期位置判断 | P1 | 中 |
| 置信度评估 | P2 | 中 |

#### 技术要求
- 算法：希尔伯特变换（Python scipy）
- 可视化：Recharts
- 更新频率：每天03:00

#### 算法逻辑
```
输入：过去24小时新闻数据（每小时计数）
处理：
  1. 希尔伯特变换提取瞬时相位
  2. 计算相位速度和振幅
  3. 预测下一个峰值时间
输出：
  - 当前相位（度）
  - 振幅强度
  - 相位速度（度/天）
  - 距周期中点天数
  - 预测峰值日期
```

## 三、技术架构

### 3.1 前端
- **框架**: Next.js 14 + React 18
- **样式**: Tailwind CSS
- **图表**: Recharts
- **部署**: Vercel

### 3.2 后端
- **API路由**: Next.js API Routes
- **数据源**: 
  - Polymarket API
  - Yahoo Finance API
  - 自定义爬虫
  - GDELT API

### 3.3 数据处理
- **定时任务**: OpenClaw Cron
- **算法**: Python (希尔伯特变换)
- **存储**: JSON文件

### 3.4 自动化任务
