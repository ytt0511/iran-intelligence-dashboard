# 伊朗情报看板管理规范

## 项目信息
- **项目名称**: iran-intelligence-dashboard
- **管理级别**: 受保护项目
- **最后更新**: 2026-03-03

## 安全防护措施

### 1. 禁止直接修改的文件
以下文件/目录只能通过自动化任务修改，禁止手工直接编辑：

| 路径 | 说明 | 修改方式 |
|------|------|----------|
| `/data/news.json` | 新闻数据 | 定时任务自动生成 |
| `/data/polymarket.json` | Polymarket数据 | 定时任务自动生成 |
| `/data/assets.json` | 资产数据 | 定时任务自动生成 |
| `/data/phase-analysis.json` | 相位分析数据 | 每天03:00自动更新 |

### 2. 允许修改的文件
以下文件可以手工修改：

| 路径 | 说明 |
|------|------|
| `/app/components/` | 前端组件 |
| `/app/api/` | API路由（除数据读取外） |
| `/app/lib/` | 工具函数 |
| `/app/types/` | 类型定义 |
| `*.config.*` | 配置文件 |

### 3. 修改审批流程

#### 3.1 小修改（样式、文案）
- 直接修改
- 本地测试
- 提交部署

#### 3.2 中修改（组件逻辑、API）
- 创建分支
- 修改代码
- 本地测试
- Hourglass审批
- 合并部署

#### 3.3 大修改（架构、数据源）
- 创建变更申请
- 详细方案设计
- Hourglass审批
- 实施修改
- 全面测试
- 部署上线

### 4. 定时任务管理

| 任务名称 | 频率 | 执行者 | 说明 |
|----------|------|--------|------|
| iran-intel-officer-news | 每小时 | 子代理 | 新闻小时报 |
| iran-polymarket-officer | 每小时 | 子代理 | Polymarket数据 |
| iran-assets-officer | 每小时 | 子代理 | 资产价格 |
| iran-phase-analysis-nightly | 每天03:00 | 子代理 | 相位分析更新 |

### 5. 数据备份

- 自动备份: 每天03:30
- 备份位置: `/data/backup/`
- 保留周期: 30天

### 6. 紧急回滚

如发现误修改，执行：
```bash
# 从备份恢复
cp /data/backup/iran-news-$(date +%Y%m%d).json /data/news.json

# 重新部署
npm run build
vercel --prod
```

### 7. 变更记录

所有修改必须记录：
- 修改时间
- 修改人
- 修改内容
- 审批人（如需要）
